GemmaQuest — Design Spec

Google I/O 2026 hackathon · serverless browser RPG · on-device Gemma via WebGPU.

The one goal

Deliver a reliable 3-minute demo whose headline is: "a frontier Gemma model is generating this game live, on my GPU, with no server." The game is the vehicle; the proof is the point.

Demo script — "prove offline first"

Beat 0

(off-camera) model pre-cached in browser storage, pipeline warmed with one inference.

Beat 1

"No server": open DevTools → Network → toggle Offline (kill wifi for theatre).

Beat 2

"It's resident": realm loads; the LOCAL · GPU badge glows green — model in memory, network dead.

Beat 3

"It's alive": walk to an NPC, ask a question, tokens stream in live while offline.

Beat 4

"It keeps generating": move; biome lore writes itself as you explore.

Beat 5

Close: "No key, no server, no internet. It just plays."

Architecture — the inference seam

The game talks only to an InferenceEngine interface; it never knows which engine answers. This makes the API fallback a one-line switch instead of a rewrite.

PRIMARY

LocalWorkerEngine

Single persistent Web Worker running Gemma via transformers.js on WebGPU. One in-flight generation, serialized through a queue.

FALLBACK / TEST

MockEngine

Deterministic, zero-network. Powers the test suite and keeps the realm narrated even with no model. The ApiEngine swaps in here for hosted Gemma if the local path wobbles.

Foundation fixes baked into this build

  • No worker-reload. One persistent worker for the whole session (the old code spawned a new worker + reloaded the 1.5GB model on every biome — OOM risk).
  • Serialized requests. A GenerationQueue guarantees one generation hits the GPU at a time.
  • Clean output. Responses are assembled from streamed tokens, not generated_text (which echoes the prompt back).
  • The proof badge. LOCAL · GPU greens only when ready && device==='webgpu'.

Scope

IN

Procedural world walk · biome lore · NPC free-form streaming chat · LOCAL·GPU badge · offline cold-open · in-browser tests.

OUT

Quest/JSON verification · branching dialogue trees · multimodal input · model-selection theatre.

Success criteria

  • The 3-minute "prove offline first" run completes with no network call in the primary path.
  • LOCAL · GPU is green; NPC dialogue and biome lore stream while offline.
  • No OOM / freeze across a full run (single worker holds).
  • If the local path fails, the engine swaps to hosted Gemma with no game code churn.

In-Browser Test Suite

Executable specs. The logic tests need no model — they use the deterministic MockEngine. The live test is gated on a loaded model.
